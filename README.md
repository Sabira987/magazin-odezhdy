# Киім дүкені — WhatsApp боты

Бұл жоба екі бөліктен тұрады:
1. **Бот** — WhatsApp-та клиентпен сөйлеседі: тілді сұрайды (ҚАЗ/РУС), санатты (Әйелдер/Ерлер/Балалар), түрін (көйлек, шалбар т.б.) көрсетеді, содан кейін фото, баға, размер, санымен тауарларды жібереді.
2. **Админ панель** (`/admin`) — сіз тауарды осы жерден қосасыз/өзгертесіз, бот дәл сол деректі клиентке көрсетеді.

---

## 1-қадам. Render-ге дейін жергілікті тексеру (міндетті емес)

```bash
npm install
cp .env.example .env
# .env файлын өз мәндеріңізбен толтырыңыз
npm start
```

Сервер `http://localhost:3000` мекенжайында көтеріледі. `/admin` бетін ашып, `ADMIN_TOKEN`-мен кіріп, тауар қоса аласыз.

---

## 2-қадам. Meta for Developers жағында WhatsApp баптау

1. https://developers.facebook.com → сіздің Business/Developer аккаунтыңызбен кіріңіз.
2. **My Apps → Create App → тип: Business** (немесе бар қолданысыңызды пайдаланыңыз).
3. Қолданбаға **WhatsApp** өнімін қосыңыз (Add Product → WhatsApp → Set up).
4. **API Setup** бетінен мыналарды көшіріп алыңыз:
   - **Temporary access token** (алдымен осы да жетеді, кейін тұрақты **System User token** жасаған дұрыс)
   - **Phone number ID**
5. Осы екеуін `.env` файлындағы `WHATSAPP_TOKEN` және `WHATSAPP_PHONE_NUMBER_ID` өрістеріне қойыңыз.
6. Сол бетте **To** бөлімінде өз нөміріңізді "test recipient" ретінде қосыңыз — тестілеу үшін.

---

## 3-қадам. Render-ге деплой

1. Осы жобаны GitHub-қа жүктеңіз (жаңа repo жасап, файлдарды сол жерге қойыңыз).
2. https://render.com → **New → Web Service** → GitHub repo-ыңызды таңдаңыз.
3. Баптаулар:
   - **Build command:** `npm install`
   - **Start command:** `npm start`
4. **Environment** бөлімінде `.env.example`-дегі барлық айнымалыны қолмен қосыңыз (`WHATSAPP_TOKEN`, `WHATSAPP_PHONE_NUMBER_ID`, `WEBHOOK_VERIFY_TOKEN`, `GRAPH_API_VERSION`, `ADMIN_TOKEN`).
5. Деплой аяқталған соң Render сізге адрес береді, мысалы: `https://kiim-dukeni-bot.onrender.com`. Соны `PUBLIC_URL` айнымалысына қойып, қайта сақтаңыз (redeploy).

---

## 4-қадам. Webhook-ты Meta-ға тіркеу

1. Meta App → WhatsApp → **Configuration** бетіне өтіңіз.
2. **Callback URL:** `https://kiim-dukeni-bot.onrender.com/webhook`
3. **Verify token:** `.env`-дегі `WEBHOOK_VERIFY_TOKEN` мәні дәл сол күйінде.
4. **Verify and Save** басыңыз — сервер дұрыс жауап берсе, жасыл белгі шығады.
5. **Webhook fields** тізімінен **messages** өрісіне жазылыңыз (Subscribe).

---

## 5-қадам. Тексеру

WhatsApp-тан test нөміріне "Сәлем" деп жазыңыз — бот тіл таңдауды ұсынуы керек, содан кейін санат, түрі, тауарлар тізбегі шығады.

> ⚠️ Meta-ның даму (development) режимінде бот тек сіз қосқан "test recipient" нөмірлеріне ғана жауап бере алады. Кез келген клиентке жауап беру үшін **Business verification** өтіп, нөмірді production-ға шығару керек (Meta App Review талап етпейтін негізгі хабарлама жіберу үшін тек business verification жеткілікті).

---

## Админ панельді пайдалану

`https://kiim-dukeni-bot.onrender.com/admin` бетін ашыңыз, `ADMIN_TOKEN` мәнін енгізіңіз. Санат → ішкі санат таңдап, тауар қосыңыз/өзгертіңіз, фото жүктеңіз. Барлығы автосақталады және бот сол сәтте жаңа деректі көрсете бастайды.

---

## Ескертпелер

- **`ADMIN_TOKEN` мен `WEBHOOK_VERIFY_TOKEN`-ды тек латын әріптері мен сандардан құраңыз** (мысалы `kiim2026admin`) — кириллица HTTP тақырыптарында (header) дұрыс жұмыс істемейді.
- 24 сағаттық сессия ережесі бар: клиент өзі жазбаса, боттан оған еркін мәтін жіберу шектеулі (тек алдын ала бекітілген **template** хабарламалар арқылы). Біздің бот тек клиент жазғаннан кейін жауап беретіндіктен, бұл шектеу әдепкі сценарийге кедергі жасамайды.
- Фотолар серверде `public/uploads` қалтасында сақталады. Render-дің тегін тарифінде диск тұрақты емес (қайта деплой кезінде тазаруы мүмкін) — өндірістік қолданыс үшін кейін S3/Cloudinary сияқты сыртқы сақтауға көшіру ұсынылады.
- Instagram боты — келесі қадам, дайын болғанда айтыңыз, дәл осы серверге қосамыз (Instagram Graph API де ұқсас webhook принципімен жұмыс істейді).
